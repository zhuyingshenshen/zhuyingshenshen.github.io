基于 JavaSE+Socket 实现局域网即时通讯软件

项目背景与需求

在学习 Java 网络编程（Socket）和 Swing GUI 开发后，为了巩固核心知识点并理解 C/S（客户端 / 服务端）架构的通信原理，我设计并开发了这款局域网即时通讯软件。

核心需求如下：

  1. 极简登录：用户仅需输入昵称即可进入聊天室，无需复杂的账号密码验证。
  2. 群聊功能：支持多用户实时群聊，消息需即时推送到所有在线客户端。
  3. 在线状态：实时展示在线用户列表及人数，用户上下线时自动更新列表。
  4. 界面友好：基于 Swing 实现可视化操作界面，包含消息展示区、输入区、发送按钮及在线列表区。

技术选型与核心思路

技术栈

  • GUI 编程：Swing（实现客户端可视化界面）。
  • 网络编程：Socket + ServerSocket（基于 TCP 协议实现客户端与服务端的可靠通信）。
  • 数据存储：HashMap（服务端用于维护在线用户与 Socket 的映射关系）。
  • 多线程：独立线程处理客户端连接与消息读写，避免单线程阻塞导致的程序假死。
  • 协议设计：自定义消息类型协议（区分登录、群聊、私聊消息）。

整体架构

系统采用标准的 C/S 架构，核心交互流程如下：

  1. 服务端：监听端口 -> 接收客户端连接 -> 分配独立线程处理 -> 维护在线用户列表 -> 转发群聊消息。
  2. 客户端：登录界面输入昵称 -> 建立 Socket 连接 -> 发送登录消息 -> 进入聊天界面 -> 实时接收服务端推送的在线列表 / 群聊消息。

核心功能实现

服务端开发

服务端启动与连接监听

服务端的核心职责是监听指定端口，并为每个新连接的客户端分配一个独立的线程（ServerReaderThread）。这样可以避免单线程阻塞，支持多客户端并发通信。

消息处理线程（核心逻辑）

服务端线程负责循环读取客户端发送的数据流，根据自定义的协议类型（1 - 登录，2 - 群聊）执行不同的业务逻辑：

  • 登录消息：将用户 Socket 和昵称存入 Map，并广播更新所有客户端的在线列表。
  • 群聊消息：接收消息内容，拼装成 “昵称 + 时间 + 内容” 的格式，转发给所有在线客户端。

客户端开发

登录与连接

基于 Swing 实现登录界面，用户点击 “进入” 后，程序尝试与服务端建立 Socket 连接，并发送登录协议头（1）和昵称。

聊天界面与消息接收

聊天界面包含消息展示区（只读）、输入区和在线列表。为了防止接收消息阻塞 UI 线程，客户端同样启动了一个独立的 ClientReaderThread 来专门读取服务端的推送数据。

项目总结与扩展

核心收获

通过本次实战，我深入理解了以下核心概念：

  1. TCP 通信机制：理解了三次握手、Socket 流的阻塞特性以及如何通过 DataInputStream/DataOutputStream 进行结构化数据传输。
  2. 并发编程：掌握了多线程在网络编程中的应用，特别是服务端如何利用线程池（或简单的多线程）处理高并发连接。
  3. 协议设计：学会了在传输层之上定义简单的应用层协议（通过 int 类型区分消息体），这是构建复杂网络应用的基础。

扩展方向

目前的版本实现了基础的群聊功能，未来可以扩展以下功能：

  • 私聊功能：在协议中增加私聊标记（type 3），并在服务端根据目标用户名查找对应的 Socket 进行发送。
  • 断线重连：增加心跳检测机制，当检测到连接断开时自动尝试重连。
  • 消息持久化：将聊天记录保存到本地文件或数据库中，支持历史记录回溯。
